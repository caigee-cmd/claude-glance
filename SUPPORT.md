# Support

## 适合提什么

欢迎以下类型的反馈：

- Bug 报告
- 功能建议
- 文档修正
- 可复现的构建或运行问题

## 提交前建议

在创建 Issue 前，建议先确认：

- 你已经阅读过 [README.md](README.md)
- 你已经检查过 [CONTRIBUTING.md](CONTRIBUTING.md)
- 问题不是当前 README 中已经写明的已知限制

## Bug 报告建议包含

- macOS 版本
- Xcode 版本
- Claude Code 是否已在本机产生 transcript
- 复现步骤
- 预期行为
- 实际行为
- 如涉及 UI，附截图

## Feature Request 建议包含

- 你要解决的真实问题
- 当前替代方案
- 你期待的行为
- 为什么它适合放进 Claude Glance，而不是外部脚本或其他工具

## 响应预期

这是一个个人维护的项目：

- 不保证即时回复
- 不保证所有功能请求都会实现
- 会优先处理明确、可复现、与当前产品方向一致的问题

## PR Expectations

如果你准备提交 PR，建议：

- 先开 Issue 或在现有讨论里对齐方向
- 保持变更范围聚焦
- 补充必要的文档更新
- 如涉及 UI，附上截图或录屏
